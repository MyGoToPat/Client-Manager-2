# HiPat - Groups System (Cohort-Based Programs)

## The Problem

Currently there's no way to CREATE groups - only select from existing ones. More importantly, the architecture is wrong:

**Current:** Directives are primary, groups are just a filter
**Better:** Groups are primary, with their own directives, message boards, and programs

## Use Case: Online Program Cohorts

A mentor runs a "12-Week Shred" program:
- **Cohort 1:** Started Dec 1, 20 people (now in Week 3)
- **Cohort 2:** Started Dec 8, 5 people (now in Week 2)
- **Gym Promo Group:** 15 people, special offers

Each cohort needs:
- Same program structure but different start dates
- Separate message board / announcements
- Group-specific directives
- Progress tracking relative to THEIR start date

---

## New Data Model

### Group Types

```typescript
export type GroupType = 
  | 'program_cohort'  // Running through a structured program
  | 'custom'          // General grouping (e.g., "Morning Warriors")
  | 'promotion'       // Limited-time promotional group
  | 'organization';   // Gym/studio based group
```

### Updated Group Interface

```typescript
export interface ClientGroup {
  id: string;
  mentorId: string;
  
  // Basic Info
  name: string;
  description?: string;
  type: GroupType;
  color?: string;
  icon?: string;
  
  // Program Info (if type === 'program_cohort')
  program?: {
    templateId?: string;      // Reference to program template
    name: string;             // "12-Week Shred"
    durationWeeks: number;    // 12
    startDate: Date;          // When this cohort started
    currentWeek: number;      // Calculated from startDate
  };
  
  // Members
  clientIds: string[];
  maxCapacity?: number;       // Optional limit
  
  // Settings
  isActive: boolean;
  isArchived: boolean;
  allowNewMembers: boolean;
  
  // Stats
  memberCount: number;
  avgProgress: number;
  avgCompliance: number;
  
  createdAt: Date;
  updatedAt: Date;
}
```

### Group Message Board

```typescript
export interface GroupPost {
  id: string;
  groupId: string;
  mentorId: string;
  
  // Content
  title?: string;
  content: string;
  attachments?: {
    type: 'image' | 'video' | 'file' | 'link';
    url: string;
    name?: string;
  }[];
  
  // Settings
  isPinned: boolean;
  notifyMembers: boolean;  // Send push notification
  
  // Engagement
  viewCount: number;
  reactions: Record<string, string[]>;  // emoji -> userIds
  
  createdAt: Date;
  updatedAt: Date;
}

export interface GroupComment {
  id: string;
  postId: string;
  userId: string;
  userType: 'mentor' | 'client';
  content: string;
  createdAt: Date;
}
```

### Group-Specific Directives

```typescript
// Directives can now be scoped to a group AND relative to program week
export interface MentorDirective {
  // ... existing fields ...
  
  // NEW: Program-relative triggers
  programTrigger?: {
    week: number;           // Trigger in week X of program
    day?: number;           // Day of that week (1-7)
    event?: string;         // Additional event within that week
  };
}
```

## Key User Flows

### Flow 1: Create New Program Cohort

1. Click "+ Create Group"
2. Select "Program Cohort"
3. Name it "12-Week Shred - January 2025"
4. Set duration: 12 weeks
5. Set start date: January 6, 2025
6. Add members (or generate signup link)
7. Enable message board
8. Create group

### Flow 2: Post Announcement

1. Go to group â†’ Message Board
2. Click "+ New Post"
3. Write content, add attachments
4. Toggle "Notify members" (push notification)
5. Post

### Flow 3: Create Program-Timed Directive

1. Go to group â†’ Directives
2. Click "+ New Directive"
3. Select "Program-timed trigger"
4. Set: Week 4, Day 1, 9:00 AM
5. Configure what Pat should do
6. Save

Each member gets this directive on THEIR Week 4 Day 1, regardless of when they joined.
